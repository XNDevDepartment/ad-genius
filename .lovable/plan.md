

## Fix: AI Model Generation Failing for Child Age Ranges

### Root Cause

The Gemini API is returning `finishReason: "IMAGE_SAFETY"` and blocking the generation. The logs show this clearly:

```
"finishReason":"IMAGE_SAFETY","finishMessage":"Unable to show the generated image. 
The image was filtered out because it violated Google's Generative AI Prohibited Use policy"
```

This happens because the prompt — even though it uses child-safe clothing — still contains terms in the `STRICTLY AVOID` section like "lingerie, see-through fabric, wet/oily skin" which, when combined with a child subject (age "4-7 years"), triggers Gemini's safety filter. The juxtaposition of child age + these negative terms (even as negatives) is what flags it.

### Solution

Modify `supabase/functions/create-base-model/index.ts` to use a **completely separate, simplified prompt** for child age ranges that:

1. Removes all adult-oriented negative terms from `STRICTLY AVOID`
2. Frames it purely as "children's clothing catalog photography"
3. Uses age-appropriate language throughout
4. Adds a retry mechanism (up to 2 retries) since safety filters can be non-deterministic

### Changes

**File: `supabase/functions/create-base-model/index.ts`**

**For `generateModelWithAI` (lines 305-345):** Split into two prompt paths:

**Child prompt** (when `isChild === true`):
```
TASK: Generate a photorealistic full-body photo of a child for a children's clothing catalog.

SUBJECT:
- Gender: ${gender}
- Ethnicity: ${nationalityText}
- Age: ${ageRange}
- Height: ${height}cm
- Build: healthy, standard proportions
- Skin tone: ${skinTone}
- Hair: ${hair.length}, ${hair.texture}, ${hair.color}
- Eyes: ${eyes}

POSE & EXPRESSION:
- Pose: ${pose}, natural and relaxed
- Expression: ${expressionText}
- Hands visible and natural

OUTFIT:
- Plain gray cotton t-shirt, blue jeans, white sneakers
- No logos or graphics

SETTING:
- Solid light gray studio backdrop
- Soft even studio lighting
- Full-body framing head to toe

OUTPUT: 2048x3072, single child centered, professional catalog quality.

AVOID: multiple people, props, blurry areas, distorted anatomy, extra limbs, watermarks, text.
```

**Adult prompt** (existing prompt, unchanged).

**Retry logic:** Wrap the Gemini API call in a loop (max 2 attempts) — if the first attempt returns `IMAGE_SAFETY`, retry once since these filters are non-deterministic. On final failure, return a user-friendly error message explaining the safety filter.

**For `uploadAndProcessModel` (lines 111-121):** Apply the same child-safe prompt simplification for the background removal prompt when `isChild === true`.

**Better error message:** When the safety filter triggers after retries, return a specific message like "The AI safety filter blocked this combination. Please try adjusting the age range or other settings." instead of the generic "No image generated by Gemini".
