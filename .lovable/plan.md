

# Analysis: Outfit Swap Generating Different Pattern

## Finding: No code change caused this

I reviewed the entire `processOutfitSwap` pipeline and the recent diff. The only changes made were:

1. **Added `imageConfig`** (aspectRatio, imageSize) to `generationConfig` — this only affects output resolution and shape, not the garment reproduction
2. **Updated credit calculation** — this has no effect on image generation

The prompt template (`CATEGORY_SWAP_PROMPTS.FULL_OUTFIT`) is unchanged and correctly instructs Gemini to use the garment from IMAGE 2. The garment analysis step correctly identifies and describes the garment.

## Root Cause

This is a **Gemini model limitation**, not a code bug. Looking at the two images:
- **Input**: Small polka-dot pattern shirt dress with belt
- **Output**: Similar silhouette but with larger medallion/mandala circles instead of small polka dots

The AI model is interpreting the garment's overall shape and style correctly (button-front dress, collar, belt, similar color scheme) but failing to reproduce the exact pattern detail. This is a known limitation of generative AI — it struggles with precise pattern replication, especially from phone photos with store backgrounds and mannequins.

## What could improve results

The garment description generated by `analyzeGarment()` is what Gemini uses as text guidance alongside the image. If the description doesn't capture the precise pattern detail (e.g., "small polka dots" vs just "patterned dress"), the model may fill in its own interpretation.

One potential improvement would be to **strengthen the prompt** to emphasize exact pattern reproduction. Specifically, in the `FULL_OUTFIT` prompt template (line 151-182), add a stronger instruction like:

```
PATTERN FIDELITY - CRITICAL:
- You MUST reproduce the EXACT fabric pattern, print, and texture from IMAGE 2
- Do NOT interpret or reimagine the pattern - copy it PRECISELY
- Small dots must remain small dots, stripes must remain stripes
- The pattern scale, spacing, and colors must match IMAGE 2 exactly
```

This would be added to all category prompts in the `CRITICAL REQUIREMENTS` section.
